# 一. 应用

Claude不能把他当作一个类似于Google的搜索引擎来使用。他是一个会推理、会综合、会辩论、会基于上下文继续往下想的伙伴。

Claude不比你聪明，它面对问题有无线的耐心、可以从很多角度思考。这一切的前提是你要让它理解你的问题

同一个话题，不同问法，输出质量天差地别

> 问：什么是X
>
> 答：只是背一个答案

>问：我在做一个每次对话调用Claude二十次的工作流，prompt caching在我的场景李能不能真的省钱，具体我该怎样使用
>
>答：它跟你一起将事情想清楚



1. 让Claude反问你

   > 在你开始之前，先问我5个最重要的、能帮你吧这件事做好的 问题，等我回答完，你再开始。
   >
   > > 让Claude自己挑出它认为最关键、最缺乏的5个问题

2. 如果你不会写prompt，就让Claude帮你写

   > 给它任务描述
   >
   > > 它生成包含：角色、上下文、格式、约束的prompt
   > >
   > > 立即执行那个prompt

3. Claude总是回答的很多显得全面、免责。既浪费token也浪费你的精力，要让他精简回答

   > 永远不要用开场白、附和、或者重复我的问题开头 
   >
   > 简单问题最多用三句话回答

4. 让Claude风格参考，只会参考表层

   > 给3-5段你写过的东西，让Claude学：句子、节奏、用词、开头结尾、避免什么
   >
   > > 注意：它可以学到你怎么写、学不到你为什么这样写。适合草稿，不适合核心搞

5.  sparring partner

   > 当你处于信心十足，将要拍板的时期。不客气、不限定的攻击你思维漏洞

6. thinking partner

   > 当你处于迷茫期，刚开始想一件事。提问、反馈，陪你将问题想清楚



## 1.1 提示词

输出一句模糊的话，然后祈祷，最后纳闷输出又幻又虚。提示词不是一句话应该是一个系统。

Claude workShop



Prompt不是一句话，是一个4层系统

1. Parseable 结构

   让Claude读得懂

2. Direction 方向

   指方向：role+ 示例+ 思考过程

3. Capability 能力

   给能力：加工具比如：skill、mcp等使用技能，赋予干活的人干活能力和更大的推理预算，而不是笼统的安排工作

   >1. 长文档放在prompt顶部，问题放在底部，因为这样更容易被关注到
   >
   >2. 随着大模型能力的升级，去年的救命补丁可能成为今年的毒药
   >
   >   > 要将prompt当作代码版本控制起来，每加一条防御性指令，最好补个为什么,当模型升级的时候一条条问防御性指令：
   >   >
   >   > > 还有用吗？没用的话删掉

4. Verification 验证

   写eval，没有eval不是调用prompt是猜测

   > + control 永远改过的case，一旦运行异常，。说明你改的东西出大问题了
   > + edge 历史上挂过的bug，每条都是regression防御
   > + boundary ：该交给人或拒答的边界，测模型知不知道自己的边界

```
真正决定回答好坏的往往是第三步，但是很多人总是在第二步兜兜转转
```



## 1.2 概念

Harness:并无需将模型变得聪明，只需：在正确的时间，把正确的上下文送到模型前面，不让噪声淹死它。

每次AI使用都要理清一个事：什么事情该进Harness，什么事情不进

```
使用AI最大的问题就是，AI不清楚你要干什么，你要将你遇见问题正确处理流程携程MD文件交给他，让模型按照你给他的流程走，相当于蒸馏下自己的认知

+ skill是一个带参数的方法调用，它描述的是判断过程，这是软件工程
```

**Harness**只做一下4件事情，所以很简单，要将问题拆分细化，像流水线。每个专属工具做自己的事情，不要大而全显得笨重

1. 循环调用模型
2. 读写你的文件
3. 管理上下文
4. 执行安全策略

> claude.md太长效果反而不好，如果内容是在太多，你可以拆分出来根据路由机制按需加载，根据任务的不同，读不同的文件，随用随取，不污染上下文。

### 1.2.1 Latent 模型该干的事情

**模糊、主观、靠理解、靠经验、没有唯一标准答案**的工作。

+ 读取，理解上下文

+ 根据资料综合判断

+ 识别不同工作模式

特点：

- 每次做，结果**可以不一样**，只要合理就行
- 需要读懂语境、理解人情、综合判断、创意发挥
- 允许 “千人千解”

**适合场景**：聊天、写文案、理解语义、分析情绪、策划方案、小型人情安排、总结归纳、翻译、构思创意等。

>你让大模型给8个人根据每个人的性格和社交氛围安排一张晚宴座位。没问题，他能做的很好。
>
>> 人少、看性格、看社交氛围，**没有唯一正确答案**，怎么排都合理，大模型随便发挥就行。



### 1.2.2 Deterministic代码该干的

**确定、客观、有唯一标准答案、讲死规则**的工作。同样的输入永远得到同样的输出

特点：

- **相同输入 → 必须完全相同输出**，差一点就是错
- 靠公式、逻辑、计算、硬性约束、组合规则运行
- 要求结果**可复现、可验证、零随机**

**适合场景**：数学计算、数据统计、批量排班、组合优化、规则校验、批量表格处理、固定流程运算、大规模排序 / 分配等。

> 让大模型给800个人排座位，他会给你一份看起来完美合理，但完全错误的座位表。因为这是一个组合优化问题，属于确定性一侧

```
最差的系统就是将工作换错了边，该Latent放在Latent，该Deterministic坚决放在Deterministic
```



# 二. Claude七层

每一层解决不同维度的问题

1. 具体对话信息和需求

2. Claude.md和rules，负责告诉Claude你是谁？

3. tools和mcp 赋予动作能力

4. skills 按需加载的方法论

5. Hooks 强制执行规则的拦截器

   > 定义的规则，Claude有时候听，有时候不听，因为那只是建议，hooks是必执行的。物理成面阻断
   >
   > 运行在模型上下文之外，0上下文成本

6. subagents 隔离上下文的独立工作者

7. Verifiers 验证闭环，确保结果可信








# 三. 详细

## 3.1 提问层面



### 1. projects

**背景知识不必再讲：**建一个project，传文档+写instructions （说明）

```
我是谁？
我在做什么？
回答时候应该注意什么写在instructions 
  + 永远不要用开场白、附和、或者重复我的问题开头 
```

就像你跟一个专属助理合作，不用每次见面都重新介绍自己。

- 你先建一个 “项目文件夹”，把相关的文档（比如你做的事、行业资料）都传进去，再写清楚 “使用说明”：

  + 比如告诉它 “我是做电商运营的，正在搞 618 活动”（我是谁、在做什么）；
  + 再定规矩 “回答别绕弯，别重复我的问题，直接说重点”（回答注意事项）。

  
```
后续聊这个项目的所有问题，Claude 都能基于这个档案答，不用你每次都重复背景，省时间也省精力。
```


### 2. Artifacts

别只让 Claude 给你写文字，它还能帮你做更直观的东西，比如：

- 想要梳理活动流程，让它直接出个流程图；
- 想要分析数据，让它做个简单的互动图表（点一下能看细节的那种）；
- 想要快速搭个小网页原型，让它直接生成能点开看的小页面。
- 核心：把抽象的文字答案，变成能直接看、能用的 “实物”，不用你自己再去排版 / 画图。

### 3. Memory

跨project给 Claude 建一个 “你的专属画像库”，新chat已经知道你是谁

> 就像你常去的咖啡店，不用每次都提醒 “少糖少冰”，店员早就记下来了。

### 4. Adaptive Thinking 扩展思考模式

让你看到Claude的推理过程

+ 往往用在复杂问题的权衡、判断上。
+ 简单问题不必使用

> thinking on
>
> thinking off





## 3.2 输出层面

给Claude一个具体的角色，挂system prompt， claude 90%的能力被默认软人格挡住。

默认的Claude会找你想法中的优点来论证支撑你的想法并不能给你很实际的建议。

+ 创建过公司也失败过的老板：挑你的漏洞

  > 你的工作不是鼓励我，而是在我做出关键的决策前提醒我的漏洞

+ 魔鬼代言人：三条最现实的失败路径：只攻击你

  > 你已经做了一个决定，让他专门给你做反方辩护，三条最现实的失败路径、最被低估的风险、最不愿面对的假设。

+ 难对话陪练：你弱的话，它会反驳你

```
结束之后，再给你做总结
```

## 3.4 配置文件

### 1. Claude.md

里面写你的规矩：代码风格、写作风格、不能用的词、文件该放哪等

### 2. Prompt Caching

> 如果你的API请求中每次都带着一段巨大的、不变的上下文：一份参考文档、system prompt、一段codebase。每次调用都在为最新处理这些token付钱。
>
> 把上述内容缓存到服务端，下次命中算便宜价格

在你想要缓存的content block加上 cache_control字段
