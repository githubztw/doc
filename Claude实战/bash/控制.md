2. `$#`: 代表脚本执行时传入的参数总个数

**文本处理三剑客**：`sed`、`awk`、`grep`

+ `sed`：擅长编辑，只要**替换**或**删除** 时候调用

+ `awk`：格式化报告"和"结构化数据处理，需要**列操作**或**计算**时候调用

+ `grep`：查找时候时调用

  > **注意：**找文件、文件夹名称的时候，使用Glob



| 维度             | grep        | sed              | awk                            |
| :--------------- | :---------- | :--------------- | :----------------------------- |
| **核心用途**     | 搜索/过滤行 | 编辑/替换文本    | 格式化/分析数据                |
| **是否修改文件** | ❌ 只读      | ✅ 可用 `-i` 修改 | ❌ 只读                         |
| **自动分割字段** | ❌ 不分割    | ❌ 不分割         | ✅ 自动分割为 `$1, $2...`       |
| **正则支持**     | ✅ 强        | ✅ 强             | ✅ 强（更丰富）                 |
| **变量/计算**    | ❌ 不支持    | ❌ 极弱           | ✅ 完整编程语言                 |
| **多文件处理**   | ✅ 支持      | ✅ 支持           | ✅ 支持（更灵活）               |
| **学习曲线**     | ⭐ 平缓      | ⭐⭐ 中等          | ⭐⭐⭐⭐ 较陡                      |
| **适合场景**     | 快速查找    | 批量替换         | 统计报表、数据清洗、文档格式化 |



# `awk`专业文本处理（列提取、计算、格式化）

> `echo cat love.txt | awk '{print $2}'`
>
> + `awk` 默认以**空格**分割每行文本，分割后的片段依次叫 `$1`、`$2`、`$3`……
> + `print $2`：打印每行**第二个字段**

```bash
# 过滤CPU大于10的进程并打印用户名、CPU
ps aux | awk '$3>10 {print $1,$3,$11}'
# 求和：统计文件数字总和
cat num.txt | awk '{sum+=$1} END {print sum}'
```

| 动作        | 含义                                 | 示例                              |
| :---------- | :----------------------------------- | :-------------------------------- |
| `print`     | 打印输出（默认以空格分隔）           | `print $1, $3`                    |
| `printf`    | 格式化输出（类似 C 语言）            | `printf "%s\t%d\n", $1, $2`       |
| `next`      | 跳过当前行，处理下一行               | `if ($1 == "") next`              |
| `exit`      | 退出 awk 程序                        | `if (NR > 100) exit`              |
| `if/else`   | 条件判断                             | `if ($3 > 100) print "high"`      |
| `for/while` | 循环                                 | `for (i=1; i<=NF; i++) sum += $i` |
| `数组`      | 关联数组（类似于 Python 字典）       | `count[$1]++`                     |
| `函数`      | 内置函数（length、substr、split 等） | `len = length($1)`                |



# `sed`：流编辑器

1. `sed`：流编辑器

   **替换s**

   + `sed 's/old/new/[g]'` ：用new替换old文本, g代表是否全局替换，默认只替换第一个匹配
   + `sed 's/old/new/[num]'` ：用new指定替换第num个old文本, num是一个数值，表示第几个匹配
   + `sed 's/old/new/I'` : I 表示忽略大小写
   + `sed -n 's/error/ERROR/p' log.txt`：表示标准输出被替换的行。n表示行数，不指定的时候，表示所有匹配的行

   ```bash
   # 基础替换（只替换每行第一个匹配）
   echo "hello world hello" | sed 's/hello/hi/'
   # 指定替换第2个匹配
   echo "a a a a" | sed 's/a/b/2'
   # 输出hi
   echo "Hello" | sed 's/hello/hi/I'
   
   # 从log.txt读取内容，将包含error的行中的 error 替换为ERROR，然后只输出这些被替换的行到终端（标准输出），而不会修改 log.txt 文件本身
   sed -n 's/error/ERROR/p' log.txt
   ```

   **删除操作d**：注意只是删除读出来的数据，源文件不变，需要d结尾

   + `sed 'n,md' file.txt` : 删除第n到第m行,m可以不写，`sed 'nd' file.txt`：删除第n行
   + `sed '/str/d' file.txt`： 删除包含"error"的行
   + `sed '/^$/d' file.txt`：删除空行
   + `sed '/^#/d' file.txt`： 删除以#开头的注释行
   + `sed '$d' file.txt`：删除最后一行

   **打印操作-n p**：将内容放在标准输出。需要加：-n和 p

   + `sed -n 'n,mp' file.txt` : 打印第n到第m行,m可以不写，`sed -n 'np' file.txt`：打印第n行

   + `sed -n '/error/p' file.txt`: 打印匹配error的行

   + `sed -n '/START/,/END/p' file.txt`：打印从"START"到"END"之间的所有行

   **行定位与地址范围**

   ```bash
   # 地址形式	含义
   '3d'	# 第3行
   '2,5d'	# 第2-5行
   '5,$d'	# 第5行到最后一行
   '/error/d'	# 匹配"error"的行
   '/start/,/end/d'	# 从匹配start到匹配end的范围
   '2,/error/d'	# 从第2行到匹配"error"的行
   ```

   **插入和追加：ia**

   + i：表示插入
   + a：标准追加

   ```bash
   # 在匹配包含pattern的行前插入
   sed '/pattern/i\这是新插入的行' file.txt
   
   # 在匹配包含pattern的行后追加（a）
   sed '/pattern/a\这是追加的行' file.txt
   
   # 在第3行前插入
   sed '3i\新行内容' file.txt
   
   # 在最后一行后追加
   sed '$a\文件末尾追加' file.txt
   ```

   **整行替换：c**

   ```bash
   # 将匹配"old"的整行替换为新内容
   sed '/old/c\这是全新的内容' file.txt
   
   # 将第10行替换
   sed '10c\新内容' file.txt
   ```

   **多命令组合 -e或者分号**

   ```bash
   sed -e 's/foo/bar/' -e 's/baz/qux/' file.txt
   sed -e 's/foo/bar/'; 's/baz/qux/' file.txt
   # 使用大括号对特定行执行多个命令
   sed '/error/{s/ERROR/警告/; s/critical/严重/}' file.txt
   ```

   



# 其他常识

1. `$#`: 代表脚本执行时传入的参数总个数

   + `$1、$2、$3...` 依次代表第 1、2、3... 个脚本入参

2. `cut`：按分隔符取截取指定列 / 字符
   -c 按字符切割

   -d 指定分隔符

   -f  取第几列

   ```bash
   # 按空格分割，取第2列（你之前学的awk同类）
   cat love.txt | cut -d' ' -f2
   # 取第1~5个字符
   echo "abc123456" | cut -c1-5
   # 冒号分割，取/etc/passwd用户名
   cat /etc/passwd | cut -d: -f1
   ```

4. `bc`：高精度计算器

   > `echo "scale=2; 44/7" | bc`  #  规定保留**2 位小数**，输出6.28

5. `if [ -d "$path" ]`

   | 参数      | 作用                                |
   | --------- | ----------------------------------- |
   | `-d 路径` | 判断是否为存在的目录                |
   | `-f 路径` | 判断是否为存在的普通文件            |
   | `-e 路径` | 判断文件 / 目录是否存在（不管类型） |
   | `-x 路径` | 判断文件是否有可执行权限            |

5. 长度统计

   + `expr length "字符串"`：`expr` 工具自带的长度统计功能，**只统计引号内文字本身的字符数**，不含换行符

     > echo `expr length "I like bash"`  # 11

   + `wc -c`：`wc` = word count，`-c` 统计**总字节数（包含换行符）**

     > echo "I love bash now" | wc -c

   + length：awk 内置函数，计算当前行文本的字符数量，**自动忽略末尾换行**，只统计文字本身

     > echo "I like and love python" | awk '{print length}'

# IF

```BASH
echo "Enter the name of the file"
# 读取用户键盘输入，存入变量 fname
read fname

if [ -f "$fname" ]

    echo "$fname exist"

```

# until  与 while

+ `while [ 条件 ]`：条件成立才循环

+ `until [ 条件 ]`：**条件不成立才循环**，刚好和 while 相反

任意 `until [ COND ]`= `while [ ! COND ]`（给条件加逻辑非 `!`）

```bash
x=5
until [ $x -le 0 ]  # while [ ! $x -le 0 ]、while [ $x -gt 0 ]  注意 操作符 !、le前后必须加空格，否则Shell会把[!$x 识别成一整个字符串
do
    sqr=$((x*x))
    echo "The square of $x is $sqr"
    ((x--))
done
```



# function

```bash
x=10
y=15

myFunc()
{
    local x=35
    y=40
    # 函数内任何变量只要没写 local，不管在哪赋值的，都是全局可见
    z=$(( x+y ))
    echo "The sum of $x and $y equal to $z"
}
myFunc # 调用方法，返回：The sum of 35 and 40 equal to 75
# z 定义在函数内部，函数执行完毕后全局不存在 z，$z 为空
echo "The sum of $x and $y equal to $z"  # 调用方法，返回：The sum of 10 and 40 equal to 75
```
