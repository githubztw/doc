# WPF Harness Demo — 编码规范

## 通用

| Rule | Level | Description |
| :--- | :---: | :--- |

## MVVM（Prism）

- ViewModel 继承 `BindableBase`，命令用 `DelegateCommand` / `AsyncDelegateCommand`
- View 的 DataContext 由 Prism 注入，禁止 XAML `DataContext="{Binding ...}"` 写死实例
- 区域导航：`IRegionManager.RequestNavigate(regionName, NavigationKeys.xxx)`
- 跨模块消息：优先接口 + 注入；必要时 `IEventAggregator`（事件类放 Abstractions）

```csharp
// ✅ ViewModel 构造器注入
public sealed class DashboardViewModel : BindableBase
{
    private readonly IGreetingService _greetingService;
    public DashboardViewModel(IGreetingService greetingService) => _greetingService = greetingService;
}

// ❌ 禁止
var svc = Container.Resolve<IGreetingService>();
```

## WPF UI（lepoco）

- Shell `App.xaml` 合并 `Wpf.Ui` 主题与 `Styles/HarnessTheme.xaml`
- 使用 `ui:NavigationView` / `ui:Snackbar` 等控件时保持与官方示例一致的命名空间
- 用户提示：优先 `ISnackbarService`（Infrastructure 封装），禁止随处 `MessageBox.Show`

## 配置

- Options 类：`public sealed class XxxModuleOptions`，属性与 JSON 字段 PascalCase（`System.Text.Json` 默认）
- 模块读取配置：仅通过 `IOptions<XxxModuleOptions>` 或 `IOptionsSnapshot<>`

## 日志

- 使用 `ILogger<T>`，禁止 `Console.WriteLine` 作为业务日志
- Shell 启动失败必须记录并友好提示

## 测试

- Services 层单元测试覆盖核心分支
- 架构规则变更必须同步 `ArchitectureTests`
