# WPF Harness Demo — 开发流程规范

## 十阶段流水线

| Stage | Name | Quality Gate | Output |
| ----- | ---- | ------------ | ------ |
| 1 | 需求分析 | 范围清晰、与 Wiki 术语一致 | 需求摘要 + `feature_list.json` 条目 |
| 2 | 方案设计 | 分层与模块边界已对照工程结构 | `.harness/changes/{feature}/summary.md` |
| 3 | 任务拆分 | 单任务 ≤ 4h | 任务勾选 + 依赖说明 |
| 4 | 编码实现 | 红线零违反 | 可编译解决方案 |
| 5 | 单元测试 | Services 核心路径有测试 | `dotnet test` 通过 |
| 6 | 架构约束 | NetArchTest 全绿 | ArchitectureTests 通过 |
| 7 | 本地验证 | `scripts/verify.ps1` | 0 error |
| 8 | 集成冒烟 | Shell 启动、区域导航 | 手动/脚本记录 |
| 9 | 文档同步 | Wiki / AGENTS 索引有效 | doc-gardening 清单 |
| 10 | 变更归档 | changes 目录完整 | summary + 回滚说明 |

## 变更目录

```plaintext
.harness/changes/{feature_name}/
├── summary.md
└── rollback.md
```

## AI 编码约束

- 单次改动不超过目标文件总行数 **40%**，超出则拆分提交
- 每次会话结束：更新 `claude-progress.txt`（若存在）与 `feature_list.json` 状态
- 编码前必读：`AGENTS.md` → 按需 `.harness/rules/`（≤3）→ `.harness/wiki/`（≤3）

## 质量检查点

| Checkpoint | Pass Criteria |
| :--------: | :-----------: |
| `dotnet build` | 0 error |
| `dotnet test` | 全通过 |
| ArchitectureTests | 0 violation |
| 模块配置 | 每模块有 `module.settings.json` 且 Shell 可覆盖 |
