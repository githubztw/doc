# 领域术语

> TulingShop 电商系统的统一术语表。编码和文档中必须使用以下术语，禁止自造术语。

------

## 核心术语

|    术语    |    英文     |                  定义                  |                   代码中的体现                   |
| :--------: | :---------: | :------------------------------------: | :----------------------------------------------: |
|    商品    |   Product   |      SPU 级别的商品，对应一组规格      |             `Product`、`product` 表              |
|    规格    |     SKU     |    商品的具体规格组合，最小售卖单位    |          `ProductSku`、`product_sku` 表          |
|   购物车   |    Cart     |          用户选购商品的暂存区          |                `Cart`、`cart` 表                 |
|  购物车项  |  CartItem   |        购物车中的一个 SKU 条目         |            `CartItem`、`cart_item` 表            |
|    订单    |    Order    |           用户提交的购买请求           |               `Order`、`order` 表                |
|   订单项   |  OrderItem  |    订单中的一个 SKU 条目（含快照）     |           `OrderItem`、`order_item` 表           |
|    库存    |    Stock    |             SKU 的可售数量             |      `stock` 字段、`StockDeductionConsumer`      |
| 金额（分） |    Cents    |         所有金额以分为单位存储         |           `priceCents`、`amountCents`            |
|   软删除   | Soft Delete | 通过 `deleted_at` 标记删除，不物理删除 |          `deletedAt`、`deleted_at` 字段          |
|    快照    |  Snapshot   |  下单时锁定商品信息，不受后续变更影响  | `OrderItem` 中的 `productName`、`unitPriceCents` |
|    分类    |  Category   |                商品分类                |                `categoryId` 字段                 |

------

## 状态术语

|  术语  |        枚举值         |             说明              |
| :----: | :-------------------: | :---------------------------: |
| 待支付 | `PENDING_PAYMENT (0)` |   订单已创建，等待用户支付    |
| 已支付 |      `PAID (1)`       |        用户已完成支付         |
| 已发货 |     `SHIPPED (2)`     |          商家已发货           |
| 已完成 |    `COMPLETED (3)`    |        用户已确认收货         |
| 已取消 |    `CANCELLED (4)`    | 订单已取消（超时 / 用户主动） |

------

## 技术术语

|    术语     |                    说明                     |
| :---------: | :-----------------------------------------: |
| Cache Aside | 缓存策略：先读缓存，未命中时读数据库并回填  |
|  Lua 脚本   |        Redis 原子操作，用于库存扣减         |
|  延迟消息   |     RocketMQ 定时消息，用于订单超时取消     |
|    幂等     | 同一操作多次执行结果一致，Consumer 必须实现 |
|  分布式锁   |       Redis 锁，防止并发操作同一订单        |
|  前向兼容   |         数据库变更不破坏旧代码运行          |
|    超卖     |        库存被超额扣减，核心防护场景         |

## 命名约定

|       模式       |          说明          |               示例               |
| :--------------: | :--------------------: | :------------------------------: |
|    `XxxCents`    |    金额字段，单位分    | `priceCents`、`totalAmountCents` |
|     `XxxVO`      |   视图对象，API 响应   |      `ProductVO`、`CartVO`       |
|     `XxxDTO`     | 数据传输对象，API 请求 |  `CartAddDTO`、`OrderCreateDTO`  |
|    `XxxEvent`    |       MQ 消息体        |       `OrderCreatedEvent`        |
|   `XxxMapper`    |  MyBatis-Plus Mapper   |         `ProductMapper`          |
|   `XxxService`   |      业务服务接口      |         `ProductService`         |
| `XxxServiceImpl` |      业务服务实现      |       `ProductServiceImpl`       |
| `XxxController`  |      REST 控制器       |       `ProductController`        |

------

### MQ Topic 命名

|        Topic        |           含义            |
| :-----------------: | :-----------------------: |
|   `ORDER_CREATED`   |        订单已创建         |
|   `ORDER_TIMEOUT`   |       订单支付超时        |
|  `ORDER_CANCELLED`  |        订单已取消         |
| `PAYMENT_COMPLETED` |        支付已完成         |
|    `STOCK_SYNC`     | 库存同步（Redis → MySQL） |

------

### Redis Key 命名

|             Key 模式              |       含义       |
| :-------------------------------: | :--------------: |
|     `tulingshop:product:{id}`     |   商品详情缓存   |
|    `tulingshop:stock:{skuId}`     |   SKU 库存计数   |
|    `tulingshop:cart:{userId}`     |    购物车缓存    |
| `tulingshop:order:lock:{orderId}` | 订单操作分布式锁 |
