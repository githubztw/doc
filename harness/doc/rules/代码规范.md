# 编码规范

## 通用规范

|        Rule         | Level |                         Description                          |
| :-----------------: | :---: | :----------------------------------------------------------: |
| 价格字段 Integer 分 |   🔴   | 所有金额以分为单位存储为 Integer，禁止 Float/Double/BigDecimal |
|     禁止魔法值      |   🔴   |     禁止代码中直接出现未解释的常量，必须定义常量类或枚举     |
|      空值处理       |   🟡   |     所有外部输入必须做空值检查，使用 Optional 或提前返回     |

------

## 后端规范

### 异常处理

- 🔴 必须使用 `BusinessException` 体系，禁止在 Controller 直接抛出非业务异常
- 异常体系：`BaseException` → `BusinessException` → `ParameterException` / `AuthException` / ...
- 全局异常处理器 `GlobalExceptionHandler` 统一捕获，返回标准错误响应
- 禁止 catch 后空处理或仅 `printStackTrace`

```java
// ✅ Correct
throw new BusinessException(ErrorCode.PRODUCT_NOT_FOUND, productId);

// ❌ Wrong
throw new RuntimeException("Product not found");
```

------

### Controller 规范

- 🔴 必须使用构造器注入，禁止 `@Autowired` 字段注入

```java
// ✅ Correct
@RestController
@RequestMapping("/api/v1/products")
@RequiredArgsConstructor
public class ProductController {
    private final ProductService productService;
}

// ❌ Wrong
@Autowired // 自动帮我 new 一个对象，直接拿来用！
private ProductService productService;

```

+ 参数校验使用JSR 303 注解（@NotNull，@Size，@Min等）
+ 返回值统一使用`Result<T>`包装

### Service 规范

- `@Transactional` 必须声明 `rollbackFor`

```java
// ✅ Correct
@Transactional(rollbackFor = Exception.class) //指定【哪些异常类型】会触发事务回滚。
public void createOrder(OrderCreateDTO dto) { ... }

// ❌ Wrong
@Transactional
public void createOrder(OrderCreateDTO dto) { ... }
```

- 只读操作使用 `@Transactional(readOnly = true)`
- Service 接口与实现分离，接口定义业务契约
- 复杂业务逻辑必须编写单元测试

------

### Repository/Mapper 规范

- 继承 `BaseMapper<T>`，使用 MyBatis-Plus 的 CRUD 方法
- 自定义 SQL 写在 Mapper XML 中，禁止在 Java 代码中拼接 SQL
- 复杂查询使用 `LambdaQueryWrapper`，禁止硬编码字段名
- 分页使用 `Page<T>` 对象

------

### RocketMQ 使用规范

- Consumer 必须幂等，重复消费不产生副作用

```java
// ✅ Correct - 幂等消费
@RocketMQMessageListener(topic = "order-created", consumerGroup = "tulingshop-cart-consumer")
public class OrderCreatedConsumer implements RocketMQListener<OrderCreatedEvent> {
    @Override
    public void onMessage(OrderCreatedEvent event) {
        // 幂等键：orderId + consumerGroup
        if (processLogService.isProcessed(event.getOrderId(), "cart-consumer")) {
            return; // 已处理，跳过
        }
        cartService.clearSelectedItems(event.getUserId());
        processLogService.markProcessed(event.getOrderId(), "cart-consumer");
    }
}
```

+ Producer 发送消息必须处理发送失败

+ 消息体必须包含 traceId 用于链路追踪

+ Topic 命名: `tulingshop-{module}-{event}`



### Redis 使用规范

- 🔴 Key 前缀必须为 `tulingshop:`

```java
// ✅ Correct
String key = "tulingshop:product:detail:" + productId;

// ❌ Wrong
String key = "product:" + productId;
```

- Key 命名规范: `tulingshop:{module}:{business}:{identifier}`
- 必须设置 TTL，禁止永久 Key
- 缓存更新策略：先更新数据库，再删除缓存
- 禁止使用 `KEYS *` 命令，使用 SCAN 替代
- 序列化使用 JSON，禁止 JDK 序列化

## 前端规范

### Vue 3 规范

- 🔴 必须使用 `<script setup>` 语法，禁止 Options API

```vue
<!-- ✅ Correct -->
<script setup lang="ts">
import { ref, onMounted } from 'vue'
const products = ref<Product[]>([])
</script>

<!-- ❌ Wrong -->
<script>
export default {
  data() { return { products: [] } }
}
</script>
```

- 组件命名 PascalCase，文件名 PascalCase
- Props 必须定义类型，使用 `defineProps<T>()`
- Emits 必须定义，使用 `defineEmits<T>()`

------

### API 层规范

- API 函数按模块拆分到 `src/api/{module}.ts`
- 统一使用 axios 实例，配置请求 / 响应拦截器
- 返回值类型必须定义 TypeScript 接口
- 错误处理统一在拦截器中处理，特殊错误在调用处处理

------

### Store 规范

- 使用 Pinia `defineStore`，命名 `use{Module}Store`
- State 必须定义 TypeScript 类型
- Actions 处理异步操作，Getters 派生计算值
- Store 之间可通过 `useXxxStore()` 互相引用



### 数据库规范

|      Rule       | Level |                       Description                        |
| :-------------: | :---: | :------------------------------------------------------: |
| 金额 INT/BIGINT |   🔴   |  金额字段使用 INT 或 BIGINT 存储分，禁止 DECIMAL/FLOAT   |
|     字符集      |   🟡   |                 统一 utf8mb4，禁止 utf8                  |
|      主键       |   🟡   |                  BIGINT 自增或雪花算法                   |
|     软删除      |   🟡   |           使用 `is_deleted` 字段，禁止物理删除           |
|    索引命名     |   🟡   | `idx_{table}_{columns}`，唯一索引 `uk_{table}_{columns}` |

------

### API 设计规范

- RESTful 风格，资源名用复数
- 版本控制: `/api/v1/{resources}`
- 请求方法: GET 查询、POST 创建、PUT 更新、DELETE 删除
- 统一响应格式: `{code: number, message: string, data: T}`
- 分页参数: `page`（从 1 开始），`size`（默认 20）
- 错误码: 4xx 客户端错误，5xx 服务端错误
