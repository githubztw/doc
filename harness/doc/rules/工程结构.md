# 工程结构规范

## 前后端分离项目目录结构

### Vue 前端（`tulingshop-frontend/`）

```plaintext
src/
├── api/         # API 请求封装
├── components/  # 通用组件
├── composables/ # 组合式函数
├── layouts/     # 布局组件
├── router/      # 路由配置
├── stores/      # Pinia 状态管理
├── styles/      # 全局样式
├── types/       # TypeScript 类型定义
├── utils/       # 工具函数
└── views/       # 页面视图
```

### Spring Boot 后端（`tulingshop-backend/`）

```plaintext
src/main/java/com/tulingshop/
├── config/           # 配置类
├── controller/       # 控制器层
├── service/          # 服务接口
│   └── impl/         # 服务实现
├── repository/       # 数据访问层
├── model/
│   ├── entity/       # 数据库实体
│   ├── dto/          # 数据传输对象
│   └── vo/           # 视图对象
├── mq/
│   ├── producer/     # 消息生产者
│   └── consumer/     # 消息消费者
└── common/
    ├── exception/     # 异常定义
    ├── enums/        # 枚举类
    ├── constant/     # 常量定义
    └── util/         # 工具类
```



### 四层架构

```plaintext
Controller → Service → Repository → Model
```

|   Layer    |        Responsibility        |      Key Annotation      |
| :--------: | :--------------------------: | :----------------------: |
| Controller | 请求路由、参数校验、响应构造 |    `@RestController`     |
|  Service   |      业务逻辑、事务管理      |        `@Service`        |
| Repository |      数据访问、SQL 映射      | `@Mapper` / `BaseMapper` |
|   Model    |         数据结构定义         |   `@TableName` / POJO    |

**依赖方向**：`Controller → Service → Repository → Model`，严禁反向依赖。

### 前后端文件放置规则

|  File Type  |      Frontend Path       |              Backend Path               |
| :---------: | :----------------------: | :-------------------------------------: |
| 页面 / 视图 |  `src/views/{module}/`   |                    -                    |
|  API 封装   |  `src/api/{module}.ts`   |                    -                    |
|  状态管理   | `src/stores/{module}.ts` |                    -                    |
|   控制器    |            -             |  `controller/{Module}Controller.java`   |
|  服务接口   |            -             |     `service/{Module}Service.java`      |
|  服务实现   |            -             | `service/impl/{Module}ServiceImpl.java` |
|  数据访问   |            -             |    `repository/{Module}Mapper.java`     |
|    实体     |            -             |      `model/entity/{Entity}.java`       |
|     DTO     |            -             |      `model/dto/{Entity}DTO.java`       |
|     VO      |            -             |       `model/vo/{Entity}VO.java`        |

### 命名规范

|      Type       |        Convention         |         Example          |
| :-------------: | :-----------------------: | :----------------------: |
|   Java Class    |        PascalCase         |     `ProductService`     |
|   Java Method   |         camelCase         |        `getById`         |
|   Java Field    |         camelCase         |       `productId`        |
|  Java Package   |       all lowercase       | `com.tulingshop.product` |
|  REST Endpoint  |        kebab-case         |  `/api/v1/product-skus`  |
| Database Table  |        snake_case         |      `product_sku`       |
| Database Column |        snake_case         |       `product_id`       |
|  Vue Component  |        PascalCase         |    `ProductCard.vue`     |
| Vue Composable  | camelCase with use prefix |       `useCart.ts`       |
|   Pinia Store   | camelCase with use prefix |    `useCartStore.ts`     |
| TypeScript Type |        PascalCase         |        `Product`         |

### 红线规则（不可违反）

1. 🔴 包名必须为 `com.tulingshop`，禁止其他包名
2. 🔴 四层架构依赖方向不可反转，Controller 禁止直接访问 Repository
3. 🔴 跨模块调用必须通过 Service 接口，禁止直接依赖其他模块的 Repository
4. 🔴 前端 API 调用必须通过 `src/api/` 封装，禁止在组件中直接发请求
5. 🔴 数据库操作必须通过 Repository/Mapper，禁止在 Service 中写 SQL
6. 🔴 新增文件必须按上述目录结构放置，禁止随意创建包 / 目录
