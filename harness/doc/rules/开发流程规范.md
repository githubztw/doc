### 开发流程规范

#### 十阶段流水线

| Stage | Name     | Quality Gate                 | Output              |
| ----- | -------- | ---------------------------- | ------------------- |
| 1     | 需求分析 | 需求清晰无歧义，业务价值明确 | 需求文档 + 影响分析 |
| 2     | 方案设计 | 方案经过评审，技术选型有依据 | 技术方案 + 变更记录 |
| 3     | 任务拆分 | 任务粒度 ≤ 4h，依赖关系明确  | 任务列表 + 执行顺序 |
| 4     | 编码实现 | 遵守编码规范，红线零违反     | 可编译代码          |
| 5     | 单元测试 | 覆盖率 ≥ 80%，核心逻辑 100%  | 测试报告            |
| 6     | 代码评审 | 评审通过，无 🔴 问题          | 评审记录            |
| 7     | 集成测试 | 接口联调通过，数据流正确     | 集成测试报告        |
| 8     | 预发验证 | 预发环境验证通过             | 验证记录            |
| 9     | 上线部署 | 部署脚本执行成功，冒烟通过   | 部署记录            |
| 10    | 线上观测 | 监控无异常，业务指标正常     | 观测报告            |

------

#### 回滚路线表

|  Stage   |     Rollback Strategy      |     Trigger      |
| :------: | :------------------------: | :--------------: |
| 编码实现 | `git checkout --` 丢弃变更 |     编译失败     |
| 单元测试 |     修复测试或回退代码     | 测试失败率 > 20% |
| 代码评审 |        修改评审意见        |   🔴问题未解决    |
| 集成测试 |    回退到上一个稳定版本    |     接口不通     |
| 预发验证 |       回退到生产版本       |     功能异常     |
| 上线部署 |     回滚到上一版本镜像     |     冒烟失败     |
| 线上观测 |      紧急回滚 + 降级       |   错误率 > 1%    |

### 变更管理目录结构

```plaintext
.harness/changes/
└── {feature_name}/
    ├── summary.md         # 变更概述
    ├── db-migrations.sql # 数据库变更
    └── rollback.sql       # 回滚脚本
```

每个变更必须包含 `summary.md`，记录：

- 变更概述
- 影响模块
- 数据库变更
- API 变更
- Redis 变更
- 关键约束
- 回滚方案

### AI 编码约束

#### 40% 阈值

- AI 单次生成代码变更量不超过文件总量的 40%
- 超过40%必须拆分为多次操作
- 每次操作后必须验证编译/运行状态

### 质量检查点

| Checkpoint | Check Content | Pass Criteria |
| :--------: | :-----------: | :-----------: |
|  编译检查  |  代码可编译   |    0 error    |
|  规范检查  | 符合编码规范  | 0 🔴 violation |
|  测试检查  | 单元测试通过  | 覆盖率 ≥ 80%  |
|  评审检查  | 代码评审通过  |   0 🔴 issue   |
