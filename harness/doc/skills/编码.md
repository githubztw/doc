## 后端编码步骤

### Step 1：Model

- 定义 Entity（对应数据库表）
- 定义 DTO（接收请求参数）
- 定义 VO（返回响应数据）
- 检查：字段类型、金额字段使用 Integer、命名规范

### Step 2：Repository

- 创建 Mapper 接口，继承 `BaseMapper<T>`
- 复杂查询编写 Mapper XML
- 检查：禁止硬编码字段名、优先使用 `LambdaQueryWrapper`

### Step 3：Service

- 定义 Service 接口
- 实现 ServiceImpl
- `@Transactional` 必须声明 `rollbackFor`
- 检查：业务逻辑完整性、异常处理

### Step 4：MQ（如需）

- 定义消息体 `Event` 类
- 创建 `Producer` 发送消息
- 创建 `Consumer` 消费消息（**必须幂等**）
- 检查要点：
  - `Consumer` 必须实现幂等性
  - 处理消息发送失败场景（如重试、补偿机制）
  - 确保 Topic/Tag 命名符合规范

#### Step 5：Controller

- 创建 Controller，构造器注入
- 定义 API 端点
- 添加参数校验注解
- 检查：返回 `Result<T>` 统一响应、参数校验完整

#### Step 6：Config（如需）

- 添加必要的配置项
- 定义 Redis Key
- 定义 MQ Topic
- 检查：Redis Key 前缀为 `tulingshop:`

------

### 前端编码步骤

#### Step 1：Types

- 定义 TypeScript 接口 / 类型
- 与后端 DTO/VO 对齐
- 检查：字段类型一致、金额字段为 number（单位：分）

#### Step 2：API

- 在 `src/api/{module}.ts` 中封装 API 函数
- 定义请求参数和响应类型
- 检查：统一使用 axios 实例、类型定义完整

#### Step 3：Store

- 创建 Pinia Store `use{Module}Store`
- 定义 State、Getters、Actions
- 检查：State 必须定义类型、异步操作统一放在 Actions

#### Step 4：Composable

- 创建组合式函数 `use{Feature}`
- 封装可复用逻辑
- 检查：composable 仅包含纯逻辑、无副作用

#### Step 5：Component

- 创建 Vue 组件，使用 `<script setup>` 语法
- 定义 Props/Emits 类型
- 检查：`<script setup>` 语法合规、命名规范正确

#### Step 6：View

- 创建页面视图
- 组合 Components 和 Composables
- 配置路由
- 检查：页面逻辑完整，数据流向清晰
