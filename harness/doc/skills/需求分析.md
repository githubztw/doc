# 需求分析技能

## 概述

将模糊需求转化为清晰的技术任务，识别业务影响和技术风险。

## 触发条件

当收到新功能需求或变更请求时，自动激活此技能。

## 执行步骤

### Step 1：需求解析

- 提取核心功能点
- 识别功能边界
- 明确验收标准
- 输出：需求清单

### Step 2：业务影响分析

- 识别受影响的业务模块
- 评估业务流程变更
- 确定用户影响范围
- 输出：影响模块列表

### Step 3：技术影响分析

- 评估数据库变更需求
- 评估 API 变更需求
- 评估缓存 / MQ 变更需求
- 评估前端变更需求
- 输出：技术变更清单

### Step 4：冲突检测

- 检查与现有功能的冲突
- 检查与编码红线的冲突
- 检查与架构约束的冲突
- 输出：冲突报告

### Step 5：输出

生成需求分析报告，写入 `.harness/changes/{feat-name}/summary.md`，包含：

- 需求描述与验收标准
- 业务影响范围
- 技术变更清单
- 冲突与风险
- 任务拆分建议

## 约束

1. 🔴 需求分析必须覆盖所有受影响模块，禁止遗漏跨模块影响
2. 🔴 任何与红线规则的冲突必须明确标注，不可跳过
3. 🟡 模糊需求必须请求澄清，禁止自行假设后推进
